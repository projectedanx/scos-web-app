Product Requirements Document: Chrome Extension with Modern TypeScript Architecture
1. Problem Statement

Development teams building Chrome extensions face architectural complexity when integrating modern frontend frameworks with Manifest V3 constraints. The service worker execution model terminates background contexts after 30 seconds of inactivity, requiring extensions to externalize state to chrome.storage rather than in-memory variables. Content scripts operate in isolated JavaScript contexts separate from page scripts, necessitating explicit communication bridges. Traditional web application patterns (persistent WebSocket connections, global state managers) require fundamental redesign for MV3 compliance.

1. Understand the fundamental architectural change

webRequestBlocking allowed imperative, runtime‑driven request blocking with arbitrary logic (e.g., inspect response body, apply ML models, block based on dynamic heuristics). declarativeNetRequest is purely declarative: you predefine static rules at extension install/update, with no runtime introspection beyond predefined conditions (domain, URL pattern, resource type).
developer.chrome
+1

Key limitations for ad blockers:
arxiv
+1

Max 30,000 dynamic rules across 30 rulesets (5 rulesets static + 30 dynamic)

Max 50,000 static rules per ruleset

No access to request/response body (cannot block based on content)

No JavaScript execution per request (no custom logic)

Rules must be bundled or fetched at install/update, not generated dynamically

Chrome's explicit intent: MV3 makes sophisticated ad blocking harder to prevent fingerprinting and performance issues.
reddit
+2

2. Choose your migration strategy

Strategy A: Pure declarativeNetRequest (Recommended for simple blockers)

Predefine ad server domains, URL patterns in static rulesets

Bundle common ad networks (Google Ads, Facebook, etc.) in static rules

Allow users to add custom dynamic rules via options page

Strategy B: Hybrid approach (Advanced)

Use declarativeNetRequest for high‑volume domains

Use offscreen documents (MV3 feature) for complex logic on low‑volume edge cases

Implement user‑customizable rulesets that download on install/update

Strategy C: Pivot to cosmetic filtering

Block ads via CSS injection (content scripts + declarativeContent) instead of network requests

Faster, less detectable by sites, but doesn't save bandwidth

3. Implement declarativeNetRequest rules
Step 3.1: Add required permissions
json
{
"permissions": [
"declarativeNetRequest"
],
"host_permissions": [
"https://*/*",
"http://*/*"
]
}


Note: host_permissions scope the rules to matching origins.

Step 3.2: Define static rulesets (bundled with extension)

Create rules.json in your extension root:

json
[
{
"id": 1,
"priority": 1,
"action": { "type": "block" },
"condition": {
"initiatorDomains": ["doubleclick.net", "googlesyndication.com"],
"resourceTypes": ["sub_frame", "script", "image"]
}
},
{
"id": 2,
"priority": 1,
"action": { "type": "block" },
"condition": {
"urlFilter": "||adservice.google.com^",
"resourceTypes": ["xmlhttprequest"]
}
}
]


Load in service worker:

javascript
chrome.runtime.onInstalled.addListener(async () => {
await chrome.declarativeNetRequest.updateStaticRulesets({
removeRuleSetIds: ['ads_rules'],
addRulesets: [{
id: 'ads_rules',
path: 'rules.json',
enabled: true
}]
});
});

Step 3.3: Implement dynamic user rules (limited to 30,000 total)

Service worker:

javascript
// Update user rules from storage or remote URL (on install/update)
async function updateUserRules() {
const rules = await getUserAdRules(); // From chrome.storage or fetched URL
await chrome.declarativeNetRequest.updateDynamicRules({
removeRuleIds: rules.map(r => r.id),
addRules: rules.slice(0, 30000) // Max 30k
});
}


rules.json format for dynamic rules:

json
[
{
"id": 1001,
"priority": 1,
"action": { "type": "block" },
"condition": {
"urlFilter": "||example-ads.com/*",
"resourceTypes": ["image", "script"]
}
}
]
