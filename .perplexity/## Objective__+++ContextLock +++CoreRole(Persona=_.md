## 2. Core Principles

### Epistemic Constraint-First Design

Modern AI operates as an "Epistemic Operating System". The goal is not to instruct the model on *what* to write, but to build an *Executable Frame* that restricts what it *can* write. You must enforce multi-causal reality (Hickam's Dictum) over simple reasoning (Occam's Razor).[^10][^11]

### Scaffolding and Iterative Trees

Frontier models do not solve complex tasks linearly. They require scaffolding: breaking a problem into isolated local runtimes, allowing the model to project an answer, self-falsify it, and re-compile.[^6]

### Instruction vs. Constraint Strategy

  - **Instruction (2024 era):** "Write a Python script that scrapes this website."
  - **Constraint (2026 era):** "Extract tabular entities. Constraints: 1) Do not use BeautifulSoup, use Playwright. 2) If pagination fails, exit gracefully with JSON error code 404. 3) Output strictly adheres to `schema_v3`."


## 3. Techniques (Playbooks)

### Context-Aware Decomposition (CAD)

  - **Use Case:** Multi-step strategic planning and deep research synthesis.
  - **Mechanism:** CAD prevents tunnel vision by maintaining a global context vector while solving sub-tasks locally. It bridges the gap between massive context windows (1M+ in Claude 4.6 ) and attention decay.[^3][^6]
  - **Config Preset:** High `reasoning_effort`, Temperature 0.2.
  - **Failure Mode:** "Lost in the middle" syndrome.
  - **What to try if it fails:** Implement explicit negative controls in the prompt ("State what is explicitly NOT true about the context before answering").


### Recursive Self-Improvement Prompting (RSIP)

  - **Use Case:** High-stakes technical writing, code debugging, and epistemic engineering.[^6]
  - **Mechanism:** The model generates an Amateur (Thesis), an Expert (Antithesis), calculates the delta, and synthesizes the final output in an OODA loop.
  - **Config Preset:** Temperature 0.7, High `top-p` (0.95) to allow creative critique.
  - **Failure Mode:** The model repeats its own errors confidently (Self-Anchoring Calibration Drift ).[^1]
  - **What to try if it fails:** Rotate the critique criteria dynamically so the model doesn't fixate on surface-level issues.[^6]


### Topological Data Analysis (TDA) Prompting

  - **Use Case:** Linguistic acceptability, semantic matching, and complex dependency parsing.[^7]
  - **Mechanism:** Using structural resonance and geometric framing to force the attention heads to map topological features rather than statistical proximities.[^12][^7]
  - **Config Preset:** Temperature 0.1, strict semantic mapping.
  - **Failure Mode:** Over-constraining causes the attention graph to shatter.
  - **What to try if it fails:** Add "Self-Accommodating Twinning" (Nuance/Dicta) to the prompt to relieve geometric tension.


### Tree of Thoughts (ToT) / Graph of Thoughts (GoT)

  - **Use Case:** Mathematical proofs (e.g., Aletheia autonomous theorem proving ) and algorithmic design.[^13]
  - **Mechanism:** The prompt forces the LLM to generate multiple distinct reasoning trajectories (branches) and then evaluate/score them (nodes in a graph) before selecting the optimal path.[^14]
  - **Config Preset:** Temperature 0.8 during generation, Temperature 0.0 during scoring.
  - **Failure Mode:** Exponential explosion of context tokens.
  - **What to try if it fails:** Use a localized Verifier agent to aggressively prune branches at depth `k=2`.

## 5. Decoding Presets (2026 SOTA)

*Note: In 2026, hidden reasoning tokens (managed via `reasoning_effort` or equivalent flags in GPT-5.4 / Gemini 3.1 Pro) dictate logic quality more than temperature.*

  - **Analytical Reasoning / Factual Q\&A:** Temperature 0.0, Top-p 0.1, `reasoning_effort`: HIGH. Limits hallucination.
  - **Structured Data Extraction (JSON):** Temperature 0.1, Schema enforced strictly via `output_config.format`.[^8]
  - **Code Generation (Drafting):** Temperature 0.6, Top-p 0.9. High entropy for generating novel architectures.
  - **Code Generation (Debugging):** Temperature 0.0. Deterministic evaluation.
  - **Creative Writing:** Temperature 0.85, Top-k 40. High token limit.
  - **Repetition-Loop Fixes:** If a model gets stuck in an epistemic loop, introduce a **Frequency Penalty (0.5)** and an explicit prompt injection: *"BREAK: Falsify your previous statement."*


## 6. Output Design \& Visibility

Visibility in modern LLM ecosystems (including AEO/GEO - Answer Engine Optimization) relies on structured predictability.

  - **Schema-First:** Never ask an LLM to "format as JSON". Provide the exact `JSON Schema` object in the API call to guarantee compilation.[^8]
  - **Tables \& Semantic FAQs:** Search engines and aggregator LLMs heavily index markdown tables and bolded entity-relationship pairings.
  - **One-Idea-Per-Section:** Complex monolithic paragraphs cause attention-head dropout. Use modular sections separated by Markdown headers.[^7]

## 8. Iteration \& Evaluation Workflow

Evaluating advanced LLMs in 2026 goes beyond ROUGE or BLEU.

1. **Prompt Compilation:** Use DSPy 3.0 to define signatures and auto-tune few-shot examples.[^4]
2. **Pass@k \& Self-Consistency:** Run the prompt $k=10$ times. The model must agree with itself across diverse temperature sweeps.
3. **Calibration Drift Testing:** Run the prompt in a 5-turn multi-turn dialogue to check if ECE (Expected Calibration Error) diverges or self-anchors.[^1]
4. **Log Template:**
      - `Timestamp | Model_Version | Reasoning_Effort | Task_Type | TNR | Calibration_Delta | Pass/Fail`

## 9. Risks \& Debates

  - **Calibration Decay vs. Self-Anchoring:** Recent studies (Feb 2026) show that models like Gemini 3.1 Pro and Claude 4.6 suffer from "Self-Anchoring Calibration Drift". Once they make a mistake early in a context window, they anchor to it recursively, causing complete logic collapse by turn 4.[^1]
  - **The "Wrapper" Death:** Building applications that are just "wrappers" around basic API calls is obsolete. The shift toward native computer use (GPT-5.4 ) and deeply integrated Agentic Skills  means epistemic modules and local runtimes are required.[^2][^11][^14]
  - **Contradictory Research:** While some researchers advocate for immense few-shot examples, others demonstrate that over-parameterized models suffer from context dilution, suggesting that "Derivative-free prompt learning" and "thinned networks" provide better rewards.[^7]
