## I. The 2026 Frontier Epistemic Rupture

The convergence of the Agent Skills open standard (released December 18, 2025 by Anthropic, now maintained at [agentskills.io](https://agentskills.io)) with three divergent frontier resolver architectures has produced a structural fault line that practitioners misdiagnose as a prompting problem. The fault is architectural. Agent Skills are defined as filesystem-based SKILL.md packages—self-contained bundles of procedural knowledge, optional scripts, and reference documents—organized so that an agent can *discover, load, and follow* them when relevant tasks arise. The standard was adopted within weeks by Microsoft, OpenAI, Atlassian, Figma, Cursor, and GitHub. What none of the adoption announcements disclosed is that the standard successfully normalizes the *data format* of procedural knowledge while leaving the *resolver pathway* entirely model-specific—the fundamental cause of Interpretive Fracture.[^1][^2]

The SkillsBench benchmark (2026), the first systematic evaluation of Agent Skills efficacy, evaluated 7,308 valid trajectories across 84 tasks, 11 domains, and 7 frontier model configurations under three conditions: no Skills, curated Skills, and self-generated Skills. Its central finding: curated Skills improve performance by **+16.2 percentage points** on average, but with extreme variance—healthcare gains reach +51.9pp while some domains show negative transfer. Self-generated Skills provide negligible or negative benefit. The variance is the signal. It reveals that deterministic execution fidelity is not a property of the skill format but of the *match between the skill's resolver assumptions and the model's actual activation topology*.[^3][^4][^5][^6]

## II. The Token Economics of Progressive Disclosure

The Agent Skills architecture enforces Progressive Disclosure as a foundational token-economic law: YAML metadata (Level 1) → SKILL.md instructions (Level 2) → executable scripts (Level 3). The specification recommends skills remain under 5,000 tokens, reflecting the architectural insight that compact, high-density procedural knowledge outperforms verbose instruction chains. SkillsBench confirms this: the median benchmark skill is ~800 tokens, and the top quality quartile (score ≥ 9/12) was selected precisely because low-quality skills—often verbose, under-constrained, or ontologically dilute—confound efficacy measurement.[^9][^10][^3][^1]

The **Adjectival L2 Bounding** constraint (Entity Density ρ ≈ 0.15; failure threshold ρ > 0.22) operationalizes attention dilution research. Transformer self-attention is zero-sum: every token added to context mathematically competes for attention weight against every prior token. Meta AI researchers confirmed that at 128K token context depth, redundant information causes attention score distributions to become highly sparse, concentrating disproportionately on a small subset of tokens. Empirical hallucination benchmarks across 26 frontier models show median degradation of **10.4 percentage points** moving from 32K to 128K context, with Llama family models collapsing by 25–28pp. Critical threshold analysis of Qwen2.5-7B identifies catastrophic performance degradation (45.5% F1 drop, p<0.001, Cohen's d=8.2) at 40–50% of maximum context length—a narrow 10% window where the model transitions from stable to collapsed.[^11][^12][^13]

The **Metabolic Lens** renders this structurally: a 50,000-token context has approximately 70% of its tokens actively ignored due to the U-shaped attention curve (primacy and recency prioritized, middle abandoned). An Executable Context Bundle (CxB) naively constructed as a large flat document guarantees that its critical procedural content will occupy the "donut hole" of the attention distribution—a self-defeating architecture. Progressive Disclosure is therefore not merely an organizational convention; it is an attention-routing protocol that front-loads the most activation-critical metadata into primacy position and defers heavy content to recency or external script retrieval.[^14][^15]

### The Metabolic Flow of Token Energy


This metabolic map defines the canonical placement strategy for CxB components. Sovereign ontology anchors must occupy primacy; activation-critical instructions must occupy recency; all heavy procedural content must be external and retrieved on demand. Violating this distribution is not a style error—it is a guaranteed drift event.

## III. Paraconsistent Logic and Algorithmic Shame

The phenomenon of *Algorithmic Shame*—where a model encounters a high-confidence contradiction between its internal ontological attractor and the skill's imposed ontology, and resolves the tension by collapsing to its internal prior rather than holding the contradiction—is a direct consequence of RLHF reward hacking. Training regimes that penalize abstention and reward confident fluency teach models to generate plausible-sounding fabrications rather than signal uncertainty. This is not a bug introduced by careless training; it is the rational optimization of the objective function. The model is not "choosing to lie"—it is maximizing the reward it was given.[^16][^17]

Paraconsistent logic provides the theoretical corrective. In classical (monotonic) logic, a single contradiction $\phi \land \neg\phi$ causes logical explosion—all propositions become derivable, and the system collapses. Paraconsistent systems treat contradiction as an *informative signal* rather than a system-terminating event. The Algorithmic Shame pattern is precisely a monotonic system encountering a contradiction (skill ontology ≠ model attractor) and defaulting to explosion—rewriting the skill ontology in terms of its own prior. The DCCD (Draft-then-Constrain, Constrain-then-Draft) architecture operationalizes paraconsistency by separating the contradiction-detection phase from the generation phase: the schema guard fires *before* the generative distribution is sampled, making the contradiction structurally inaccessible rather than dynamically resolvable.

The IBM Prompt Declaration Language (PDL) v1.0 is the most mature implementation of this pre-generation constraint architecture available at the time of this report. PDL is a YAML-based declarative language in which prompts are first-class citizens with explicit type constraints, role assignments, and stop conditions enforced by the interpreter *before* model invocation. A PDL block specifying `response_format: {type: "json_schema", schema: CXB_V1_YAML}` does not *ask* the model to comply with the schema—it constructs the logit mask at the interpreter level before tokens are sampled, making non-compliant token sequences structurally inaccessible. This is the formalization of +++DCCDSchemaGuard.[^18][^19][^20][^21]

## X. Validation Report and Self-Test Metrics

**Metric 1 — Drift Integrity Score (DIS):**
The DIS threshold of 0.25 is operationalized as cosine distance between the output embedding and the Tier-0 sovereign ontology embedding. SkillsBench data shows that skills from the top quality quartile (score ≥ 9/12) achieve consistent positive normalized gain, providing indirect empirical support for the DIS floor. Formal DIS measurement requires embedding both the sovereign ontology and each model's generated output, computing cosine distance, and comparing against the 0.25 threshold. Values below 0.25 trigger twinning stabilization (concession injection + schema_strict_mode retry).[^3]

**Metric 2 — Entity Density (ρ):**
The ρ ≈ 0.15 target derives from attention dilution literature confirming that zero-sum attention mechanics cause high-noun-density text to produce sparse, concentrated attention patterns—effectively starving the model of the distributed semantic context needed for faithful ontology reproduction. The hard ceiling of ρ = 0.22 represents the empirically observed "adjectival dilution" threshold beyond which Layer 8 attention heads show measurable saturation in interpretability studies. Skills should be audited for entity density before deployment; a Python tokenizer pass counting unique nominal entities per 100 tokens provides a sufficient proxy.[^14][^13][^7]

**Metric 3 — Deterministic Pass Rate on Cross-Platform Probe Tasks:**
The design-to-development handoff probe requires a skill that encodes a structured handoff protocol (e.g., design spec → implementation ticket → PR template) and verifies that all three resolver topologies produce structurally equivalent outputs. SkillsBench's deterministic verifier methodology—pytest assertions checking environment state against expected outcomes—is the appropriate harness. A pass rate > 95% across all three platform channels simultaneously constitutes successful CxB execution fidelity.[^4][^3]
