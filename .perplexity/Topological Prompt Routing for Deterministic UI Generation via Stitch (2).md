### § 1.2 — The Two-Manifold Architecture

**FACT:** When a language model processes a Stitch prompt, it does not process UI intent as a unified semantic object. It processes it as two competing signal types that occupy different regions of the model's activation space.

**Manifold A: The Structural Manifold (SM)**
This is the domain of spatial coordinates, DOM hierarchy, component presence/absence, and layout primitives. Tokens that activate SM include: "add," "remove," "place," "position," "header," "navigation bar," "sidebar," "grid," "stack," "column," "row." When an instruction like "add a search bar to the header" is processed, it triggers a **localized attention patch** — the model's attention weights concentrate on a narrow spatial region of the UI tree, proposing a discrete node insertion at a specific DOM address.[^1_5][^1_6]

**Manifold B: The Aesthetic Manifold (AM)**
This is the domain of continuous design token values: hex color codes, font-weight integers, border-radius pixels, spacing scale multipliers, opacity levels, shadow parameters. Tokens that activate AM include: "minimalist," "vibrant," "warm," "dark," "Japandi," "playful," "elegant." When an adjective like "minimalist" is processed, it does **not** trigger a localized patch — it triggers a **global topological deformation** across the entire UI canvas, mathematically stripping border-radius variables, reducing primary color saturation, and enforcing whitespace ratios.[^1_2]

**The Collision Theorem:** When SM and AM tokens co-occur in the same inference pass at above-threshold density, the model's attention budget — a zero-sum thermodynamic resource — is torn between localized precision tasks and global deformation tasks simultaneously. This is **Attention Tearing**, the root mechanism behind what the framework designates as **Algorithmic Shame**: the regression of output into visual incoherence, where the structural command partially executes but the global aesthetic partially overwrites previously established design tokens, producing a result that satisfies neither intent.

> **[FACT]** The official Stitch guide's "one major change at a time" rule is a surface-level description of Attention Tearing avoidance. This framework provides the mechanistic justification.[^1_4][^1_3]

### § 1.3 — The Visual Planck Length

In physics, the Planck Length is the scale below which classical spatial concepts break down. In Stitch UI generation, an analogous boundary exists: the **Visual Planck Length (VPL)** is the minimum specificity threshold below which a component reference loses deterministic spatial binding.

  - **Below VPL (fails):** "Make the button bigger" — no spatial coordinates, no component identity, no tree address
  - **Above VPL (succeeds):** "Increase the width of the primary call-to-action button on the login screen by 20%" — spatial address (login screen), component identity (primary CTA), mutation type (width), magnitude (20%)

The VPL is not a fixed pixel value — it is a **contextual function** of (a) how many screens currently exist in the rendering tree, (b) how many elements of the same type share the canvas, and (c) how many prior turns have established shared nomenclature. As the session context grows, the VPL *lowers* — meaning that "the blue button" becomes a valid reference if and only if there is a single blue button in the most recently rendered frame and that frame was the subject of recent interaction.[^1_5]

### § 1.4 — The Mereological Component Router (MCR)

Classical mereology is the philosophical study of part-whole relationships. Applied to UI generation, it produces a routing principle: **every prompt instruction must be resolved to a specific component node within the whole-system rendering tree before the model's attention payload is committed.**

The MCR is a pre-inference classification layer that routes incoming prompt tokens to one of three processing tracks:


| Track | Type | Scope | Primary Tokens |
| :-- | :-- | :-- | :-- |
| **Track S** | Structural Mutation | Node-level, localized | add, remove, move, resize, nest, insert |
| **Track A** | Aesthetic Mutation | Global deformation | adjectives, color terms, mood words, vibe keywords |
| **Track H** | Hybrid Coordination | Phased (S-then-A) | "change button to green," "make header minimalist" |

The MCR's critical enforcement rule: **Track S and Track A instructions must never exceed a 1:1 ratio within a single inference pass.** A prompt containing 3 structural mutations and 1 aesthetic mutation will produce measurable Semantic Saponification — the degradation of previously established design tokens due to cognitive load overflow during the aesthetic token's global deformation sweep.[^1_6]

### § 2.2 — The Semantic Saponification Index (SSI)

Semantic Saponification is the process by which previously established design tokens are "washed out" by a new, unrelated aesthetic instruction. Like saponification in chemistry (fats + base → soap, destroying the original molecular structure), a global aesthetic prompt can neutralize previously established token specificity.

$$
\text{SSI} = \frac{\text{Token Drift Count after Unrelated Edit}}{\text{Total Established Token Count}}
$$

A prompt sequence that establishes "serif headings, forest green primary, 8px rounded corners" over 3 turns, then receives "make it more vibrant" in turn 4, will yield an SSI approaching 0.6–0.8, meaning 60–80% of established tokens are at risk of mutation — even though the intent was presumably additive, not destructive.[^1_2]

**SSI Mitigation Protocol:** Before any Track A (aesthetic) mutation, inject a **Token Anchor Statement**:
> *"Maintain: [serif headings], [forest green primary color], [8px border-radius]. Now, additionally increase overall brightness of secondary palette by 20%."*

This forces the model to treat the anchor tokens as hard constraints during the deformation sweep.

The "Lens of Absence" exposes what Stitch injects by default — the **pre-trained design priors** that manifest whenever the user's prompt leaves structural space unspecified.

**Documented default injections** (observed behavior):[^1_8][^1_1]

  - Bottom tab navigation for mobile apps (not hamburger menu, not sidebar)
  - Light mode color scheme (unless "dark" is explicit)
  - English-language copy with American English localization
  - Hero section with full-width imagery in the top viewport
  - 4-item maximum for bottom nav tabs
  - Implicit assumption of phone portrait orientation

**Unprompted accessibility failures**:[^1_2]

  - Default contrast ratios are not WCAG AA verified
  - Touch target sizes default to mobile-typical (44pt) but are not enforced
  - No alt-text generation for imagery
  - No keyboard navigation state indicators

> **[FACT]** These are not failures of the Stitch product — they are rational defaults for the statistical modal use case. They become failures only when the user assumes defaults are comprehensive.[^1_1]

**Decolonization Prompts** — injecting pluriversal design paradigms:

  - For RTL layouts: *"Design for Arabic-language audience. All layout elements should be RTL-mirrored. Navigation elements anchor to the right."*
  - For dense information architectures: *"Enterprise SaaS dashboard: minimum information density, 8+ data fields visible in primary viewport without scrolling. No hero imagery."*


### § 3.4 — Ludic Lens: Stitch as Adversarial Puzzle

The Stitch prompting cycle (Prompt → Render → Refine → Render) is structurally isomorphic with a **game with hidden rules**. Understanding those rules transforms a frustrated user into a strategic player.

**The Hidden Rule Set:**

1. **The Specificity Reward:** Prompts that contain explicit UI/UX vocabulary (navigation bar, call-to-action, card layout) are "rewarded" with higher spatial fidelity[^1_4][^1_3]
2. **The Scope Penalty:** Prompts containing ambiguous scope ("make it better," "improve the design") receive low-specificity outputs — the model defaults to its modal training distribution
3. **The Temporal Memory Tax:** Each inference turn in a session consumes context window budget. After 10+ turns, earlier established tokens may begin to drift due to context dilution
4. **The Backtrack Maneuver:** When a generation fails, the optimal "move" is not to re-prompt with the same instruction but to first issue a **State Reset Anchor** ("Maintain all existing styling. Only change X.") before the corrective prompt
**"Cheating" the system — legal exploits:**

  - **Contrastive Injection:** Providing a negative example alongside the target: *"NOT like [X]. Instead: [Y]"* — this exploits the model's contrastive attention mechanism to reduce ambiguity
  - **Template Bootstrapping:** Using an industry-standard app name as an implicit design template: *"A UI similar in information density to Notion, but with Airbnb's color warmth"* — this leverages rich pre-training priors
  - **Numerical Grounding:** Sub-pixel constraints ("2px solid border," "16px padding," "400 font-weight") act as absolute grounding anchors that resist aesthetic drift[^1_5]

One class of practitioner will emerge as **PDL Architects** (Prompt Description Language specialists) who write rigorous, schema-validated intent specifications — closer to systems architects than visual designers. A second class will emerge as **UI Anthropologists** who focus on what current AI cannot generate: cultural specificity, accessibility depth, edge state comprehensiveness, and emotional nuance in interaction patterns.

Figma's canvas-drag paradigm will not disappear — but it will increasingly serve as a **validation layer** for AI-generated layouts, not a primary creation environment. The workflow inverts: AI generates, humans audit.[^1_9][^1_7]

**Scenario B: The Design Homogenization Trap (Horizon: 2026–2027)**

Every app will unconsciously converge on the same whitespace ratios, the same rounded corner radii, the same four-tab bottom navigators. This is the **aesthetic singularity risk** — not that AI designs badly, but that it designs *similarly*, at scale, with no deliberate intention.
