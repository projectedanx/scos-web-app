## Inspector — Retry Service Network Resiliency
**Edge Case:** The retry logic handler (`executeWithRetry`) lacked coverage for exhaustion of transient network failures, default and overridden fallback values, rate limit threshold boundaries, and custom predicate overrides.
**Assertion:** Interrogated using pure Node.js test runner to mock and simulate various failure states (429 Rate limits, Network connection timeouts, 500 server errors). Bombarded the handler with successive errors to prove it correctly exhausts retries, returns fallback values, and properly escalates errors natively. We asserted exact call counts on mocked functions to prove boundaries are respected, and utilized the Sabotage Check to mathematically prove the backoff calculator function (`*=`) enforces correct delays by temporarily mutating it to (`+=`).

## Inspector — Capsule Compiler XSS & Fallback Resiliency
**Edge Case:** The UI HTML generator (`capsuleCompiler.ts`) lacked safety assertions for mitigating malicious payload injections (XSS) within dynamically bound values (Titles, Arrays, Semantic Tables). It also lacked fallback boundaries to ensure the document compiled successfully given empty or missing meta/section structures.
**Assertion:** Interrogated using pure Node.js test runner by passing malicious payloads `<script>alert(1)</script>` into string and array dependencies to verify the `escapeHtml` pure function executed correctly without causing the runtime to crash. Asserted graceful fallbacks during undefined data structures to mathematically prove empty capsules continue to render skeleton UI patterns instead of throwing native errors. Validated with a Sabotage Check mutating `escapeHtml` to return raw unencoded variables.
