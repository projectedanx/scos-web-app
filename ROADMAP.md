# 🗺️ Sovereign Cognitive OS: Strategic Roadmap

## Status Report: v1.4.0 (The Sovereign Library Era)

### ✅ Completed Capabilities (The Bedrock)

**Core Systems**
- [x] **Gemini 3/Pro Integration:** High-fidelity instruction following via `gemini-3-pro-preview`.
- [x] **Resilient JSON Engine:** Heuristic repair logic to salvage truncated or malformed LLM outputs.
- [x] **Client-Side Cryptography:** ECDSA P-256 Key generation and Manifest signing.

**Agent Forge**
- [x] **Manifest Fabrication:** Text/URL ingestion to structured JSON.
- [x] **Architectural Discourse:** Chat interface for refining agent constraints.
- [x] **Version History:** Non-linear navigation (Restored vs. Refined vs. Fabricated) with keyboard support.
- [x] **Risk Analysis:** Automatic classification of tools (Low/Medium/High/Critical).
- [x] **Interactive Filtering:** Toggle visibility of external tools based on risk severity.

**Agent Library**
- [x] **Visual Grid:** Card-based view of all fabricated agents in the Vault.
- [x] **Quick Stats:** At-a-glance metrics for Tools, Abilities, and Anchors.
- [x] **Restoration Loop:** Seamlessly load archived agents back into the Forge.

**Capsule Lab**
- [x] **Knowledge Distillation:** Turning raw text into structured `ContextCapsule` JSON.
- [x] **Artifact Minting:** Compiling capsules into standalone, Tailwind-styled HTML pages.

**Data Persistence**
- [x] **Sovereign Vault:** LocalStorage based session management.
- [x] **Import/Export:** Full JSON portability for backup and sharing.
- [x] **Provenance Indexing:** Tracking source URLs and hashes.

---

### 🚧 Immediate Priorities (The Bridge)

**Phase 2: The Uplink (Connecting to Reality)**
- [ ] **Python Swarm Connector:** Create a standard Python library (`pip install scos-core`) to load `.json` manifests generated by this app.
- [ ] **Direct Export API:** Simple local endpoint to push manifests directly to a running Python swarm instance.
- [ ] **Capsule Embedding:** Logic to automatically inject a "Context Capsule" into the system prompt of a running Agent.

**Phase 3: Governance & Evolution**
- [ ] **Manifest Diffing:** Visual UI to compare two versions of an Agent Manifest and highlight changes in constraints or tools.
- [ ] **Team Signatures:** Multi-sig support (requiring multiple Commander keys to validate a manifest).
- [ ] **Registry Sync:** syncing the local "Provenance Index" with a shared team database (PostgreSQL/Supabase).

### 🔮 Future Horizons (The Lattice)

- **WASM Sandboxing:** Execute simple Agent logic directly in the browser using WebAssembly.
- **Evolutionary Heuristics:** Allow agents to propose updates to their own source code (requires "Smooth Chromosome" governance).
- **Inter-Agent Protocol (IAP):** Standardized message bus for agents to exchange Capsules.
